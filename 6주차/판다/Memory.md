## 메인 메모리 (Main memory)
  - CPU가 직접 접근할 수 있는 기억 장치
  - 프로세스 실행을 위해 프로그램이 메모리에 적재되어야 한다.
  - 주소가 할당된 일련의 바이트로 구성

## 메모리 관리 장치 (MMU : Memory Management Unit)
  - 논리 주소를 하드웨어의 물리 주소로 변환
  - 메모리 보호/캐시 관리 등 CPU의 메모리 접근을 관리하는 하드웨어
  - 유저가 기억장소를 일일히 할당하지 않도록 해준다.
  - 프로세스가 실제 메모리의 물리적 용량을 초과해도 실행될 수 있도록 한다.
  - 레지스터와 메모리의 연산 속도 차이가 크므로 중간에 캐시가 존재 -> 자주 사용하는 데이터를 캐시에 두고, 캐시에 없는 데이터만 메모리에서 찾는다 -> 시간/공간적 지역성 고려

  ### 메모리 보호
    - 프로세스에 메모리 공간을 할당하고, 공간에 벗어난 접근 요청이 오면 거절
    - 프로세스에 할당되는 영역 설정은 커널 모드에서만 가능
    
  ### 메모리 과할당
    - 실제 메모리 용량보다 큰 용량을 프로세스에 할당한 경우
    - 페이징 시스템을 유저에게 인식시키지 않도록 페이지 스와핑을 진행해야 한다
    
  #### 페이지 교체
    - 실행 중 페이지 부재 발생
    - 메모리에 없는 페이지를 디스크에서 탐색
    - 메모리에 남는 프레임이 있는지 확인하고, 없으면 victim 프레임을 선정
    - 빈 프레임에 탐색한 페이지를 올림
